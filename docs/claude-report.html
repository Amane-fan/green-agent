01 / 问题背景

核心难点是多约束路径规划

这是一类车辆路径问题（VRP），但比普通最短路多了分类合规、容量、可达性与满溢优先级—— 收得太晚会溢出，收得太多走冤枉路，则油耗与碳排放上升。

  • 分类约束：车辆、垃圾桶、处理厂三方类别必须匹配，不能混收。
  • 时序约束：满溢率持续变化，需提前识别即将超阈值的点位。
  • 路径约束：道路图须可达，访问顺序直接影响里程、油耗与碳排放。
讲述要点：一句话点题——这是带分类、容量、可达性和满溢优先级的车辆路径问题，目标是在约束下让路线既"及时"又"低碳"。
📥输入

垃圾桶位置 / 满溢率 / 容量 / 类别、道路边权、车辆能力与油耗、处理厂接收范围。

⚙️处理

Agent 编排：预测满溢 → 筛选待收运 → 类别分组 → 可达性分析 → 车辆分配与路线优化。

📤输出

每车收运顺序、完整路径节点、总距离 / 油耗 / 碳排放，以及未分配点位的原因。

02 / 系统架构

前端可视化 · 规划服务 · 知识共享池

  • React 19 + React Leaflet：地图点位、道路边与彩色车辆路线。
  • FastAPI + SSE：场景生成、时间推进、路线规划、历史与 AI 流式接口。
  • LangGraph：串联监测 / 知识 / 编排 / 可达性 / 车辆-设施五个 Agent 节点。
  • NetworkX + KnowledgePool：完成图计算，沉淀场景、预测与规划历史。
讲述要点：关键设计——LangGraph 负责状态编排与可解释 trace，而图校验、最短路、满溢模拟、路径优化都是确定性 Python 函数；同一 seed + 场景结果可复现，便于演示与测试。
架构：前端 → FastAPI → LangGraph → 规划算法 → KnowledgePool

03 / 多智能体任务编排

LangGraph 把规划流程拆成 5 个 Agent 节点

1Monitoring 预测未来 3 步满溢率（horizon=3），筛选当前或预测 ≥ 阈值的垃圾桶。
2Knowledge 载入场景与预测时域作为共享上下文，沉淀到 KnowledgePool 供复盘。
3Task Orchestrator 按垃圾类别组织 eligible 任务，记录待收运数量。
4Reachability 构建确定性图，准备 NetworkX 可达性分析。
5Vehicle-Facility 候选筛选 + 贪心分配 + 遗传算法排序 + 处理厂选择 + 路径还原。
共享状态 PlanningState： scenariothreshold predictionseligible traceresult 每个节点向 trace 追加一条带数据的步骤 → 前端可解释"为何收 / 为何未分配"。
讲述要点：编排是一条确定性流水线：感知 → 知识 → 任务 → 可达性 → 分配，重计算集中在首尾两节点；全过程记录 trace，使决策可追溯、可解释。

04 / 路径规划实现 A

先预测筛选与车辆分配，再做路线排序

  1. 预测与筛选：当前或未来 3 步满溢率 ≥ 阈值（默认 70）的垃圾桶进入待收运集合。
  2. 优先级：按满溢率从高到低处理，优先解决高风险点位。
  3. 候选车辆：类别匹配、车→桶→处理厂均可达、剩余容量足够。
  4. 分配策略：字典序最小化 (任务数, 距离, 随机扰动)，兼顾负载均衡。
讲述要点：这一步是"两阶段优化"的第一阶段——把哪些桶交给哪辆车定下来，下一页再优化单车的访问顺序。
# 候选车辆需同时满足 vehicle.supported_waste_type == bin.waste_type nx.has_path(vehicle.node_id, bin.id) nx.has_path(bin.id, facility.node_id) remaining_capacity >= estimated_volume
estimated_volume = round(bin.capacity × fill_rate / 100, 3) selected = min(candidates, key=(任务数, 距离, random))

07 / AI 规划助手

对规划结果"问得出、答得准"

  • 上下文接地：把完整规划记录（路线 / 指标 / trace / 场景快照）序列化注入系统提示。
  • 流式输出：OpenAI 兼容接口 + SSE 逐字返回，前端 Markdown 渲染。
  • 护栏：助手"只解释、不修改路线、不编造数据"；规划结果变化即清空对话。
  • 安全：API Key 仅后端持有；缺配置返回 503，不影响核心功能。
讲述要点：这是把"黑盒优化"变"可对话"的一层——比如问"为什么这辆车路线最长"，助手只基于当前 trace 与指标作答。

09 / 创新点与难点

关键设计取舍

🧩两阶段优化

贪心分配（哪辆车）+ 遗传算法排序（什么顺序）解耦，约束清晰、易解释。

🔍可解释失败

未分配任务归因 unreachable / capacity / category，而非简单丢弃。

🌱碳感知指标

距离→油耗→碳排放贯穿规划，紧扣"绿色低碳"主题。

🕸️连通性保证

难点：随机连边易产生孤立点。方案：近邻主干 + 75% 概率补边，先连通再加密。

🎯满溢优先 vs 里程

难点：最短未必最优。方案：适应度内置延迟惩罚，让高满溢桶尽早被收。

🗺️重叠路线可视

难点：多车共用道路重叠。方案：二次贝塞尔曲线偏移，分流不串色。

讲述要点：挑一个最有共鸣的难点讲透即可，例如"连通性保证"或"满溢优先与里程的权衡"。

10 / 总结与展望

从静态展示到可解释、可复现、可优化

绿运先锋把垃圾分类收运推进为一套多智能体调度流程：感知满溢、按类编排、约束分配、进化优化、 还原路径并量化碳排放，全程可追溯、可对话、可回溯。

讲述要点：用一句话收束当前成果，再自然过渡到"如果继续做，会往哪走"。

后续可拓展方向

🛰️真实路网与实时数据

接入 OSM 真实路网与 IoT 满溢传感，用真实行驶距离替代坐标近似。

⚖️多目标优化

里程 / 碳排放 / 及时性帕累托前沿，让决策者按偏好选解。

🧠动态重规划

引入强化学习与在线重调度，应对临时新增任务与车辆故障。
