01 / 问题背景

分类收运的核心难点是“多约束路径规划”

  • 不同类别垃圾不能混收，车辆和处理厂都存在类别匹配约束。
  • 垃圾桶满溢程度随时间变化，系统需要提前识别即将达到阈值的点位。
  • 城市道路抽象为图结构，车辆必须能从当前位置到达垃圾桶，再到达处理厂。
  • 路径顺序影响总里程、油耗和碳排放，是本项目重点优化对象。
讲述重点：先说明这是一个车辆路径问题，但比普通最短路多了分类、容量、可达性和满溢优先级。
输入

垃圾桶位置、满溢率、容量、垃圾类别、道路边权、车辆能力、处理厂接收范围。

处理

Agent 编排任务，筛选待收运垃圾桶，做可达性分析并规划每辆车路线。

输出

车辆收运顺序、完整路径节点、距离、油耗、碳排放和未分配原因。

02 / 系统架构

前端可视化 + 后端规划服务 + 知识共享池

技术栈与模块

  • React + React Leaflet：展示地图点位、道路边和车辆路线。
  • FastAPI：提供场景生成、时间推进、路线规划和历史记录 API。
  • LangGraph：串联监测、知识、编排、可达性、车辆处理厂匹配 Agent。
  • NetworkX：道路图建模、连通性校验和最短路径计算。
  • KnowledgePool：保存满溢历史、预测结果、规划结果和场景快照。

03 / 多智能体任务编排

LangGraph 将规划流程拆成 5 个 Agent 节点

Agent 1 Monitoring 计算未来 3 步预测，筛选当前或预测满溢率超过阈值的垃圾桶。
Agent 2 Knowledge 加载当前场景和预测结果，为后续规划提供共享数据。
Agent 3 Task Orchestrator 按垃圾类别组织待收运任务，准备车辆分配。
Agent 4 Reachability 基于道路图准备可达性分析，过滤不可达点位。
Agent 5 Vehicle Facility 执行车辆分配、路径排序和处理厂选择，输出最终路线。
START → monitoring → knowledge → task_orchestrator → reachability → vehicle_facility → END
讲述重点：每一步都会写入 trace，前端可以解释“为什么这些垃圾桶被收运、为什么有些未分配”。

04 / 路径规划实现 A

先做任务筛选与车辆分配，再做路线排序

  1. 预测满溢：根据当前满溢率和增长趋势计算未来 3 步预测值。
  2. 筛选任务：当前或预测值超过阈值，例如 70%，进入待收运集合。
  3. 排序优先级：待收运垃圾桶按满溢率从高到低处理。
  4. 车辆候选：必须类别匹配、路径可达、剩余容量足够。
  5. 分配策略：优先选择任务数更少、距离更近的车辆，兼顾负载均衡。
候选车辆条件： vehicle.supported_waste_type == bin.waste_type has_path(vehicle.node_id, bin.id) has_path(bin.id, compatible_facility.node_id) remaining_capacity >= estimated_volume
estimated_volume = bin.capacity × fill_rate / 100 vehicle = min(任务数量, 到垃圾桶距离, 随机扰动)

05 / 路径规划实现 B

重点：用遗传算法优化单车收运顺序

  • 当某辆车分配到超过 2 个垃圾桶时，进入遗传算法排序。
  • 初始种群包含 1 条贪心路径和 23 条随机排列，兼顾质量和搜索空间。
  • 每代保留适应度最好的 8 条路线作为父代。
  • 交叉方式为“保留左父代前缀 + 按右父代顺序补齐剩余节点”。
  • 25% 概率执行两点交换变异，避免陷入局部最优。
  • 迭代 35 代后选择适应度最低的顺序作为收运顺序。

06 / 输出指标与可视化

从“收运顺序”还原为可展示的完整道路路径

  • 路径序列为：车辆当前位置 → 多个垃圾桶 → 兼容处理厂。
  • 相邻节点之间调用 NetworkX 最短路径，拼接成完整 `path_node_ids`。
  • 距离累加为路线总里程，油耗按车辆 `fuel_per_km` 估算。
  • 碳排放按 `estimated_fuel × 2.31 kg/L` 估算。
  • 无法分配的任务给出原因：不可达、容量不足或类别不匹配。

07 / 工程保障与演示价值

系统不仅能规划路线，还能解释和复现实验结果

可达性分析

道路图先做连通性校验；分配前使用 `nx.has_path` 检查车辆、垃圾桶、处理厂之间是否可达。

知识共享池

保存 `fill_history`、`predictions`、`planning_records`，支持历史路线恢复和结果复盘。

测试覆盖

后端测试覆盖类别约束、不可达任务、容量限制、高满溢优先级和 API 编排。

课堂演示

随机生成场景、推进时间、规划路线、查看历史记录，形成完整演示闭环。

08 / 总结

围绕路径规划完成了一个可演示的多智能体系统

  • 用 LangGraph 将监测、知识、任务编排、可达性和车辆规划拆成清晰 Agent 流程。
  • 用 NetworkX 把道路转为图，支持连通性检查和最短路径查询。
  • 路径规划重点采用“约束分配 + 遗传算法排序”，兼顾满溢紧急度和行驶距离。
  • 前端地图展示车辆路线、指标和历史记录，满足课程系统演示要求。
结束语：本项目把垃圾分类收运从静态地图展示推进到可解释、可复现、可优化的调度系统。
