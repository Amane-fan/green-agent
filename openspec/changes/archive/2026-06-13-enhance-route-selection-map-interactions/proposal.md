## Why

当前地图会一次性渲染所有已规划车辆路线，自定义节点也通过自动坐标偏移来放置，这会降低路线查看和场景编辑的精确性。演示需要更清晰的路线聚焦交互：选择一条车辆路线，只在地图上查看该高亮路线，在该路线上下文中查看路段指标，并通过点击目标地图位置来放置自定义节点。

## What Changes

- 让右侧规划面板中的路线卡片可被选择。
- 默认选中规划结果返回的第一条路线。
- 地图上只将选中的车辆路线渲染为高亮路线图层。
- 恢复 React Leaflet 瓦片背景图层，并将其置于场景标记、图边和高亮路线之后。
- 点击普通图边时展示基础图边长度。
- 仅在点击高亮路线段时展示所选路线段长度和车辆相关燃油信息。
- 将自定义节点创建从自动偏移放置改为两步流程：先选择节点类型，再点击应创建节点的地图位置。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `route-visualization`：新增选中路线的地图高亮、可选择的路线卡片、瓦片背景渲染，以及路线段指标弹窗。
- `map-scenario-management`：新增通过地图点击放置自定义垃圾桶、车辆和处理设施的能力。

## Impact

- 影响 `frontend/src/App.tsx` 中的前端 React 地图渲染。
- 影响 `frontend/src/styles.css` 中的前端样式。
- 影响 `frontend/src/App.test.tsx` 中的前端路线和地图交互测试。
- 预计不需要修改后端 API 或数据模型；路段燃油可由所选路线的车辆数据和图边权重推导得到。
